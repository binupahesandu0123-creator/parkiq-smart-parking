🔒 Privacy & Security
How ParkIQ protects user accounts, QR access, wallet payments, and parking data
🛡️ Protection Summary SECURE
🔐 Account Access
Login Protection
Only authenticated users and authorized admins can access system features.
📱 QR Safety
Unique Tokens
Entry and exit QR tokens are unique per user and can be regenerated anytime.
💳 Payment Security
Wallet + Card Controls
Payments are validated before processing and recorded for transaction tracking.
📜 Audit Trail
History Logs
Parking sessions, payments, and account activity are traceable through the system history.
Privacy Policy Highlights
1. Personal data collection
ParkIQ stores only the information needed to deliver the smart parking service, such as user name, email, phone number, vehicle number, vehicle type, wallet balance, parking session data, and payment history.
2. Purpose of data usage
User data is used to authenticate accounts, generate QR access, assign parking slots, calculate rates, process payments, provide receipts, and improve parking recommendations.
3. Limited data sharing
Personal and vehicle information should only be visible to the account owner and authorized system administrators for operational and support purposes.
4. User control
Users can review their wallet activity, session history, and QR details inside the system and regenerate QR codes if misuse is suspected.
Security Controls
🔑 Authentication
Separate user and admin login areas reduce unauthorized access to management functions.
🧾 Secure payment flow
Payment validation, wallet balance checks, and transaction logs help prevent incomplete or invalid payments.
🚧 Access restriction
Admin features remain hidden unless an admin signs in, protecting sensitive operational data.
⏳ Session traceability
Entry time, exit time, parking area, slot, duration, and receipts help track every parking session clearly.
